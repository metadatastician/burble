== Sync procedure

GitHub wikis are themselves a git repo: `.wiki.git` under the same
organisation. The simplest manual sync:

[source,bash]
----
# one-time
git clone https://github.com/hyperpolymath/burble.wiki.git ~/burble.wiki

# each sync
cp docs/wikis/Home.md         ~/burble.wiki/Home.md
cp docs/wikis/Quickstart.md   ~/burble.wiki/Quickstart.md
cp docs/wikis/Architecture.md ~/burble.wiki/Architecture.md
cp docs/wikis/Glossary.md     ~/burble.wiki/Glossary.md
cd ~/burble.wiki && git add -A && git commit -m "sync: from main" && git push
----

A future `just wiki-sync` recipe will automate this.

== Conventions

* GitHub wikis use Markdown by convention (not AsciiDoc), so wiki
  source files here are `.md`. The directory README itself is
  `.adoc` to match the rest of `docs/`.
* File name = wiki page name. Spaces become `-` in the URL
  (`Foo Bar.md` → `wiki/Foo-Bar`).
* Don't duplicate full doc content here — the wiki *signposts* the
  canonical docs in the repo. Long form belongs under `docs/`.
* When the wiki page diverges from this source, the source wins —
  next sync overwrites the wiki.

This directory holds Burble's long-form documentation. Source-code-level
docs live next to the code they describe (`server/`, `ffi/zig/`, `src/`,
etc.) — anything that survives a refactor independent of those modules
lives here.

The taxonomy follows the RSR (Rhodium Standard Repository) template's
pillar / unit pattern, augmented for Burble's voice-platform domain.

== Layout

[cols="1,3", options="header"]
|===

== Conventions

* AsciiDoc (`.adoc`) is preferred for new docs; existing `.md` files are
  left as-is unless rewriting.
* Every directory has a `README.adoc` (top-level) plus a numbered
  `0.X-AI-MANIFEST.a2ml` (machine-readable). Date-stamped reports
  follow the pattern `<topic>-YYYY-MM-DD.adoc`.
* Cross-link with `link:` (AsciiDoc) or relative URLs (Markdown). Don't
  hard-code `https://github.com/...` to repo files; relative links
  follow renames.

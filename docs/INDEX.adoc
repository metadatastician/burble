=== link:architecture/[`architecture/`]

  cross-project integration record

=== link:decisions/[`decisions/`]

Architecture Decision Records (ADRs). One file per decision, numbered
sequentially, append-only. See `decisions/README.adoc` for the index.

Currently:

* `0001` — Adopt RSR standard
* `0002` — IndieWeb-native optional integration
* `0003` — PAKE + SAS tiered auth
* `0004` — Bolt QUIC dual-bind
* `0005` — WSL2 Bolt inbound UDP without mirrored networking
* `0006` — `proven-frame` not applicable to Bolt packet
* `0007` — Claims-to-evidence discipline
* `0008` — Formal-proof enforcement vs scope (Option C)

== Machine-readable docs

Burble carries a parallel machine-readable layer for tooling /
agents. None of it is hidden from humans, but it's not where humans
read first.

* link:../.machine_readable/[`.machine_readable/`] — top-level
  contractiles, configs, AI manifests, agent instructions
* `0-AI-MANIFEST.a2ml` at the repo root — entry point for any agent
* `0.X-AI-MANIFEST.a2ml` in every doc subdir — directory-scoped
  manifest with `X` = depth from root
* `*.a2ml` companion files alongside human-readable docs
* `contractiles/` (symlink → `.machine_readable/contractiles/`)
* `schemas/config.cue` — CUE schema for runtime config

If you're adding documentation: AI manifests are auto-generated by
the contractile tooling, not hand-edited.
